# tailwindcss变量如何设置

## 原理
tailwindcss是使用rust实现的。
nextjs项目通常会自带tailwindcss依赖，它会被一个名为`postcss.config.mjs`的文件进行引入，该文件会通知nodejs。
会在开发者每次启动pnpm run dev的时候开始启动一个rust服务，rust服务直接根据项目的根目录开始进行查找css文件，直到找到引入tailwindcss的文件为止。
- 第一步：tailwindcss会开始仔细解析我们引入tailwindcss的css文件`一般是global.css`，它会启动一个ast转换服务，将所有读取到的css转换为拆建需要的json结构，在这之中，我们注册的一些tailwindcss变量就会被保存起来。
- 第二步：tailwindcss会迅速扫描所有的前端源文件，如果有hmr的话则读取更新的文件，并将这些文件的css内使用到的tailwindcss变量进行转义，转回我们在文件内声明的css变量。像：`background-color: var(--color-nav-bg);`
- 第三步：tailwindcss会在最终生成的css最终产物内，加上所有提及的tailwindcss变量，没有使用或者提及的则不会加入其中，形成普通css方式的映射

**疑问**
你可能会有疑问，为什么之前的tailwindcss使用的是类似tailwindcss.config.ts的方式来声明变量的方式，为什么现在tailwindcss转而直接在css文件内进行读取？
在v3版本的时候，tailwindcss在每次解析的时候必须重新启动一个v8进程去解析配置文件，而且这个文件可能还存在有其他的依赖进行引入，导致每次读取配置文件产生非常大的消耗
但是在v4版本，使用了rust进行重写后，rust读取这种静态的css文件是非常快速的，它只需要建立ast树并进行读取出使用到的css即可。

## 引入tailwindcss
我们需要在入口文件引入一个css文件，通常是`global.css`文件
接下来在第一行加入`@import "tailwindcss";`
```css
@import "tailwindcss";
```
一个tailwindcss的配置文件包括有以下内容：
- root 原生的css变量
- @theme inline  告诉tailwindcss引擎，这里面的将是内联的变量
```css
@import "tailwindcss";

:root {
  --background: #ffffff;
  --foreground: #171717;
}

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --font-sans: var(--font-geist-sans);
  --font-mono: var(--font-geist-mono);
}

@media (prefers-color-scheme: dark) {
  :root {
    --background: #0a0a0a;
    --foreground: #ededed;
  }
}

body {
  background: var(--background);
  color: var(--foreground);
  font-family: Arial, Helvetica, sans-serif;
}

```
这就是一个完整的tailwindcss变量。不过我们需要知道一些前缀:
- color：代表这是颜色
- font：代表这是字体
- spacing:代表这是宽度
