# 接入 Better Auth

接下来是比较简单的一步，也是上一步的延续。绝大多数网页都需要实现登录功能。

我选择使用 [Better Auth](https://www.better-auth.com/) 来实现登录功能。这是一个对 Next.js 以及个人网站开发非常友好的身份验证库，可以迅速接入项目。并且登录、登出、获取用户信息等功能都已直接集成，无需二次开发。

---

## 基本操作

### 1. 安装依赖

```bash
npm install better-auth
```

### 2. 设置环境变量

主要在 `.env` 或 `.env.local` 中设置以下几个核心参数：

- **BETTER_AUTH_SECRET**：`better-auth` 的密钥。可以直接在 Better Auth 官网生成，也可以在终端使用 `openssl rand -base64 32` 生成。
- **BETTER_AUTH_URL**：`better-auth` 的主要访问地址（通常是本地的 `http://localhost:3000`）。

```env
BETTER_AUTH_SECRET=your_generated_secret_here
BETTER_AUTH_URL=http://localhost:3000
```

### 3. 创建服务端实例

在项目中创建两个文件，分别用于服务端和客户端。

首先创建服务端配置文件 `lib/auth.ts`，用于对接数据库。如果使用 Prisma，可以直接引入 `prismaAdapter` 并在 `betterAuth` 中进行配置：

```ts
import { betterAuth } from "better-auth";
import { prismaAdapter } from "better-auth/adapters/prisma";
// 如果你的 Prisma 生成路径在其他地方，请根据实际情况修改
import { PrismaClient } from "@/generated/prisma/client";

const prisma = new PrismaClient();

export const auth = betterAuth({
  database: prismaAdapter(prisma, {
    provider: "mysql",
  }),
});
```

直接引入 `prismaAdapter` 并在配置项中注册即可。这代表 `better-auth` 已经完成了与后端数据库的连接。

### 4. 数据库表初始化（生成与迁移）

连接上数据库后，我们还需要在数据库中创建相应的表结构。可以通过以下命令自动生成 `better-auth` 所需的数据库表并运行迁移：

```bash
npx auth@latest generate
npx auth@latest migrate
```

### 5. 接入第三方账号登录（如 GitHub）

目前我们没有密码登录等复杂的需求，只需要接入常见的第三方社交登录即可。在 `lib/auth.ts` 中补充 `socialProviders` 配置：

```ts
import { betterAuth } from "better-auth";

export const auth = betterAuth({
  // ... 其他配置（如 database 等）
  socialProviders: { 
    github: { 
      clientId: process.env.GITHUB_CLIENT_ID as string, 
      clientSecret: process.env.GITHUB_CLIENT_SECRET as string, 
    }, 
  }, 
});
```

> [!NOTE]
> 这里的 `GITHUB_CLIENT_ID` 和 `GITHUB_CLIENT_SECRET` 需要前往 GitHub 开发者后台（GitHub Developer Settings）创建 OAuth App 申请获得。

### 6. 创建客户端 Client 实例

在客户端（Client Component）中发起请求、执行登录、退出等操作，都需要通过 Client 实例。在 `lib/auth-client.ts` 中进行如下配置：

```ts
import { createAuthClient } from "better-auth/react"

export const authClient = createAuthClient({
  /** 服务的基地址（如果在同一个域名下开发，可以省略此参数） */
  baseURL: "http://localhost:3000"
})
```

### 7. 在客户端执行登录与登出

在 UI 组件中，你可以直接调用 `authClient` 的方法：

**登录示例：**

```ts
const result = await authClient.signIn.social({
  provider: "github",
  callbackURL: callbackURL ?? window.location.href,
});
```

**登出（退出登录）示例：**

```ts
const result = await authClient.signOut();
```

## 进阶封装：统一的状态管理与 Hook

在实际的生产项目中，仅仅调用基础的 `signIn` 和 `signOut` 是不够的。为了保证良好的开发体验，我们通常需要解决以下三个层面的问题：

1. **服务端支持**：在 React Server Components (RSC) 中直接获取并消费 Session。
2. **客户端支持**：在 Client Components 中能够极简地调用登录、登出、加载状态等。
3. **全局状态共享 (Store)**：使用全局 Store 共享当前用户状态，避免多层组件 Prop 传递（Prop Drilling）或客户端重复发起 Session 请求。

基于以上需求，我们可以做如下封装：

### 1. 全局状态 Store (`store/user-store.ts`)

使用 Zustand 创建一个轻量级的全局 Store，用于共享当前登录用户的信息。

```ts
import { createStore, useStore } from "zustand";
import { authClient } from "@/lib/auth-client";

// 从 better-auth 中提取出 Session 的类型信息
type SessionData = ReturnType<typeof authClient.useSession>["data"];
// 提取出 User 的类型信息
export type AuthUser = NonNullable<SessionData>["user"];

interface UserState {
  user: AuthUser | null;
  isAuthenticated: boolean;
}

interface UserActions {
  setUser: (user: AuthUser | null) => void;
  clearUser: () => void;
}

// 创建全局 store
export const userStore = createStore<UserState & UserActions>()((set) => ({
  user: null,
  isAuthenticated: false,
  setUser: (user) => set({ user, isAuthenticated: user !== null }),
  clearUser: () => set({ user: null, isAuthenticated: false }),
}));

export function useUserStore<T>(selector: (state: UserState & UserActions) => T): T {
  return useStore(userStore, selector);
}
```

### 2. 统一封装 Hooks (`hooks/use-auth.ts`)

封装常用的 React Hook，方便在客户端组件中一键调用登录、退出、获取当前状态：

```ts
"use client";

import { useCallback, useState } from "react";
import { authClient } from "@/lib/auth-client";
import { useUserStore, userStore } from "@/store/user-store";

type SessionData = ReturnType<typeof authClient.useSession>["data"];
type LocalAuthUser = SessionData extends { user: infer U } ? U : null;

// 通用 auth 状态 hook
export function useAuth() {
  const { data, isPending, isRefetching, error, refetch } = authClient.useSession();
  return {
    session: data?.session ?? null,
    user: (data?.user ?? null) as LocalAuthUser,
    isAuthenticated: Boolean(data?.user),
    isLoading: isPending || isRefetching,
    error,
    refetch,
  };
}

// 获取当前用户信息
export function useAuthUser() {
  const { user, isAuthenticated, isLoading, error, refetch } = useAuth();

  return {
    user,
    isAuthenticated,
    isLoading,
    error,
    refetch,
  };
}

// 封装第三方登录逻辑
export function useAuthLogin() {
  const [isPending, setIsPending] = useState(false);

  const loginWithGithub = useCallback(async (callbackURL?: string) => {
    setIsPending(true);
    try {
      const result = await authClient.signIn.social({
        provider: "github",
        callbackURL: callbackURL ?? window.location.href,
      });
      if (result?.error) {
        throw new Error(result.error.message ?? "GitHub 登录失败");
      }
      return result;
    } finally {
      setIsPending(false);
    }
  }, []);

  return {
    loginWithGithub,
    isPending,
  };
}

// 封装登出逻辑
export function useAuthLogout() {
  const [isPending, setIsPending] = useState(false);

  const logout = useCallback(async () => {
    setIsPending(true);
    try {
      const result = await authClient.signOut();
      if (result?.error) {
        throw new Error(result.error.message ?? "退出登录失败");
      }
      userStore.getState().clearUser();
      return result;
    } finally {
      setIsPending(false);
    }
  }, []);

  return {
    logout,
    isPending,
  };
}

// 快捷获取全局状态中的当前用户
export function useCurrentUser() {
  return useUserStore((s) => s.user);
}

// 快捷获取全局状态中的是否已登录
export function useIsAuthenticated(): boolean {
  return useUserStore((s) => s.isAuthenticated);
}
```

### 3. 数据水合 Provider (`components/hydrate-user-provider.tsx`)

因为我们在 Next.js 服务端渲染（SSR）中就能拿到用户信息，我们可以通过一个轻量级组件在客户端初次渲染（或者水合）阶段直接将数据同步到 Zustand Store 中。

这样可以避免客户端再次去请求 Session 接口产生的“白屏/数据加载闪烁”现象：

```ts
"use client";

import { useRef } from "react";
import { type AuthUser, userStore } from "@/store/user-store";

interface HydrateUserProviderProps {
  user: AuthUser | null;
  isAuthenticated: boolean;
}

export function HydrateUserProvider({ user, isAuthenticated }: HydrateUserProviderProps) {
  const isHydrated = useRef(false);

  // 只在初次加载时水合一次数据，避免 React 并发模式下重复 setState 引起的警告
  if (!isHydrated.current) {
    userStore.setState({ user, isAuthenticated });
    isHydrated.current = true;
  }
  
  return null;
}
```
