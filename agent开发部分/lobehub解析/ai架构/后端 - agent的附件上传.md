# 附件预处理
可能存在两种情况，用户上传的是buffer或者url，我们在这函数内部将其抹平称buffer，为了方便后续统一处理，在这里我们需要针对文件的不同类型做一层处理：
 -  如果是buffer，则直接使用buffer
 -  如果是url，则直接调用原生fetch将其下载
 这样在后续就可以拿到统一的buffer进行后续处理

**这里有个重点，为什么会出现url**
这里是agent可以在某些场景为了优化上传体验，将图片或者附件上传到后端前，我们前端可以针对其大小做出逻辑判断，当附件的大小超出网关可以维持的阈值时可以将其上传到cdn再发给后端。
你可能会想说直接让llm读url不行吗，问题是有时候上传的url会带上鉴权问题等
