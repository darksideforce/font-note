# 沟通大模型
这一步就是真正的和大模型进行

- 进行工具的预处理，注入必备的工具以及用户提到的工具，将其提炼成一个配置表
- 将当前未提到的工具进行简略处理，处理成一个简略的配置表，
- 两者都将处理成纯文本准备发给llm
- 将本地的skills和必须准备的skills还有历史使用到的skills进行一个融合，准备发给llm
- 创建一个空的messageid，用于展位和将后续的信息拼接在这里
- 开始发布一个状态，通知前端`stream_start`，前端会在接收到这条信息后将messageid给进行一个切换
- （可选）通过rag系统，选出外挂知识库里的相关信息，准备发给后端
- 创建一个基本client，本质上是对openai sdk的一层封装，以便后续的发送chat等行为
- 创建一个buffer数组，并创建一个函数，该函数将在buffer数组被推入数据时以每x秒为间隔向前端发送事件。
- 进行错误重试，函数会在指定次数内尝试重新连接llm
- 每一步错误重试都删除buffer数组
- 对client发送一切处理好的数据，这一步就是真正的将数据提交给llm，我们还需要将一些事件的处理注册成回调交给这一步，client并不会自主处理事件的切换，只会通知agent进行到了哪个事件。
- client停止回复，触发stream_end，清空buffer数组
- 准备进入下一轮循环

## 如何控制输出的频率 #buffer
这一步很简单。我们使用新的类似节流器的方式。每次都创建一个计时器，并在指定时间后进行销毁，在这段时间内收集信息，并在销毁时进行推送数据，这样就可以保证每次推流都有数据会落入buffer数组，并且在x秒后将buffer数组推送给前端
```ts
let buffer  = [] 
let bufferTimer = null
let flushBuffer = ()=>{
  const delta = buffer
  buffer = ''
  if(!!buffer){
    //执行推流	  
  }
}
const response = await modelRuntime.chat({
	callBack:{
		onText:(text)=>{ // 注册文本回调
			buffer += text //推流
			if(!bufferTimer){ //判断本地有没有计时器
				bufferTimer = setTimeout(async()=>{
					await flushBuffer() //清空储存起来的缓存并推流
					bufferTimer = null
				},50)
			}
		}
	}
})
```
