# Drizzle 是什么以及如何在 Next.js 项目中实现引入？

Drizzle 是一种新的 ORM，它的作用和 Prisma 类似，都是通过 TypeScript 或者 JavaScript 的方式，为数据库提供一种更加语义化、更加贴近实际需求的第三方工具。我们可以使用更贴近代码的方式来访问数据库。

一般来说，TypeScript 环境下的 ORM 主要有 TypeORM、Prisma、Drizzle。三者中 Prisma 的综合能力是最为强大的，但是它也有自己的各种 API，需要学习时间，同时也存在其他问题。这里我们使用最轻量、最适合这个 Demo 项目的 Drizzle 作为 ORM。数据库使用 PostgreSQL。

## Drizzle 的引入

在引入 Drizzle 之前，我们需要知道一些背景：因为 Node.js 和 PostgreSQL 之间天然的差异，导致 Node.js 无法直接调用 PostgreSQL 的一些 API。PostgreSQL 拥有自己独特的一套协议来进行交互，所以社区诞生了两套用于 Node.js 和 PostgreSQL 交互的驱动：

- **node-postgres**：历史更久、生态更加成熟的 PostgreSQL 客户端。
- **postgres-js**：使用 JavaScript 设计，主打轻量，支持原生的 ES Module，受到 Drizzle 支持。

这两种驱动是不兼容的。ORM-驱动-数据库是一种类似于：逻辑引擎 => 连接层 => 数据库的方式。ORM 通常把底层连接细节交给 driver 处理，自己更关注 TypeScript 的对象类型约束、查询构造和结果映射。

Drizzle 目前的官方文档中，如何引入 Drizzle 的实例也有两种方式的区别：这里我们只讲一下 postgres-js 的方式：

### 1. 下载依赖

```bash
npm i drizzle-orm postgres
npm i -D drizzle-kit
```

这里的 `drizzle-orm` 是 Drizzle ORM 本体，`postgres` 是 postgres-js 这个 PostgreSQL driver。

### 2. 创建环境变量，在根目录创建

```env
DATABASE_URL="postgresql://postgres:postgres@localhost:5433/motoko_agent"
```

### 3. 在根目录引入

需要注意，Drizzle 的官方文档上并不是专门适配 Next.js 的。在非 Next.js 项目里，我们常常会在入口的 `index.js` 引入 Drizzle 的实例，但是在 Next.js App Router 项目里没有这种传统单入口文件，而且 Next.js 还有服务端组件和客户端组件的区分。
所以我们需要在根目录下创建一个 `db` 文件夹，并创建 `index.ts` 文件。

```typescript
import 'server-only';

import { drizzle } from 'drizzle-orm/postgres-js'; // 引入特定的 Drizzle 对象
import postgres from 'postgres'; // 引入 postgres 驱动

import * as schema from './schema';

const connectionString = process.env.DATABASE_URL;

if (!connectionString) {
    throw new Error('DATABASE_URL is missing in environment variables');
}

const globalForDb = globalThis as unknown as {
    conn?: postgres.Sql;
}; // 通过单例模式创建唯一的 PostgreSQL 客户端

const conn =
    globalForDb.conn ??
    postgres(connectionString, {
        max: 10,
    });

if (process.env.NODE_ENV !== 'production') {
    globalForDb.conn = conn;
}

export const db = drizzle(conn, { schema }); // 导出，这就是一个可以直接使用或者供其他组件消费的客户端
```

最后一步，我们需要给 `drizzle-kit` 提供一些配置文件，在根目录下创建 `drizzle.config.ts` 文件。

```typescript
import 'dotenv/config';
import { defineConfig } from 'drizzle-kit';

export default defineConfig({
    out: './drizzle',
    schema: './src/db/schema.ts',
    dialect: 'postgresql',
    dbCredentials: {
        url: process.env.DATABASE_URL!,
    },
});
```

### 4. 引入 Better Auth #better-auth

关于 Better Auth 的使用，前文已经赘述过，不再过多赘述。
我们直接使用 Better Auth 提供的脚手架启动安装：

```bash
npx auth init
```

也可以自己进行手动安装。我们需要安装好三个部分：数据库 schema、客户端、实例，准备好 `secret key` 和当前的 URL。
先在 `.env` 里添加上 `secret key` 和当前的 URL。

```env
BETTER_AUTH_SECRET="b3e6f0a0131292724423e82c42f1b492"
BETTER_AUTH_URL="http://localhost:3000"
```

创建 Better Auth 实例。

```typescript
// auth.ts
import { betterAuth } from 'better-auth';

export const auth = betterAuth({
    // ...
});
```

接入数据库（**注意 ⚠️，因为我们这里使用的是 ORM，需要使用专门的适配器**），这里是后端实例，提供方法供后端使用，包含最核心的数据库交互逻辑。

```typescript
// auth.ts
import { db } from '@/db'; // your Drizzle instance
import { drizzleAdapter } from 'better-auth/adapters/drizzle';
import { betterAuth } from 'better-auth';

export const auth = betterAuth({
    database: drizzleAdapter(db, {
        provider: 'pg', // or 'mysql', 'sqlite'
    }),
});
```

创建 schema，这里推荐使用官方命令来生成 schema 或者迁移文件，该命令会生成最基本的表结构。

```bash
npx auth@latest generate
```

使用数据库迁移，将数据库与我们本地的表结构进行同步。这个命令会对比数据库与我们本地定义的 schema 是否有差异，并把差异直接推送到数据库，更适合开发阶段；生产环境通常更推荐生成 migration 后审查执行。

```bash
npx drizzle-kit push
```

创建客户端，该客户端导出方法、API 等供 Web 前端使用，不包含对数据库的访问。

```typescript
// auth-client.ts
import { createAuthClient } from 'better-auth/react';

export const authClient = createAuthClient({
    baseURL: process.env.NEXT_PUBLIC_BETTER_AUTH_URL, // 确保前端也能显式指向同一个后端地址
});
```
