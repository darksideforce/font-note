# 前端的状态机是什么？
	我们需要先明白几个问题：
	- agent是一个会有多个数字刷新的架构，每次agent流式数据的更新都会引起ui的重绘，
	- agent的信息列表有很多的状态,前端无法单独追踪一个状态
	- 前端侧需要追踪每次发送行为，方便进行取消等
	- 前端需要支持多个任务并行，而且为了避免每次都需要去向后端进行请求，这些任务是需要存储在前端进程里的
	所以我们需要一个前端状态机来进行追踪变化，管理变化
# 规避重绘问题
独立出 chatStore（有operationStore） 后，UI 会通过导出的 Selector Hooks 进行精准消费。 例如需要订阅 useOperationStore(s => s.operations[opId]?.status === 'running')。大模型字吐得再快（数据面变更），只要 Operation 状态依然是 running，这个按钮组件就绝对不会**被触发无意义的重新渲染。


# 发送事件的处理
在实现这个的时候真的是碰到很多平时开发vue的时候很难碰到的问题，我先来简单说一下最终的架构。

先是有三层基础架构：chatstore- conversationstore-inputstore三种store，分别对应最外层的总store，单独用来视觉展示的store和最底层用来输入的store，这三个基础store就已经可以涵盖所有用到的数据结构了。
chatStore：负责最基本的一些属性，当前激活的对话，所有的对话集合，将消息以对话存储。还有很多负责将消息按不同的key推入指定message桶中。
conversationStore：它只需要从顶层拿到数据即可，我们通过激活的对话id来控制这个store每次都只拿到消息集合里独属于当前数据的message

```tsx
AgentChat.displayName = 'AgentChat';

```
可以看到我们是首先使用memo来包裹了数据，这样的话页面只有在message改变的时候才进行重新渲染，memo是通过props的监听来进行控制渲染的时机。
接下来将message传入到storeUpdater，这是一个无头组件，他会时刻监听message的变化，这样message的变化只会导致StoreUpdater的重新渲染
更接下来，所有的下面的组件都只通过zustand来进行订阅，只有订阅zustand的相应属性进行重新改变才会重新渲染

## 点击发送的逻辑

我们在这里很简单，
- 1：是通过inputstore（通过另一个provider注入）来拿到输入内容。
- 2：清空输入内容
- 3：生成临时的消息，分别对应user和assistant，使用的都是临时id
- 4：使用store将临时消息推进全局message，此时会被同步到conversationStore
- 5：ui接收到状态改变，拿到新数据，更新ui
- 6：请求后端，后端生成真实的一系列id，并直接返回全量的消息数组
- 7：直接调用replaceMessage，使用全量的message，开辟一个新的message桶
- 8：调用switchTopic，切换到新的topic，此时store指向了新的topic，但是实际上数据的key并没有改变，所以ui并不会大规模重绘

```ts
export const chatAichatConversation: sliceCreator<ChatConversation, chatStore> = (set, get, store) => {
    return {
        sendMessage: async () => {
            const editor = get().editor;
            if (!editor) {
                console.warn("Editor controller not found in chatStore!");
                return;
            }
            const content = editor.getContent();
            if (!content.trim()) return;
            editor.upContent("");
            // 从 Store 获取当前的会话 ID (如果没有，请务必去 chatTopic slice 里补上 activeTopicId 这个 state)
            const topicId = (get() as any).activeTopicId || null;
            const tempUserId = `tmp_user_${Date.now()}`
            const temAssistantId = `tmp_assistant_${Date.now()}`
            get().optimisticCreateMessage(
                {
                    topicId: topicId || '',
                    title: '',
                    description: '',
                    status: 'success',
                    role: 'user',
                    content
                },
                {
                    tempMessageId: tempUserId,
                    // operationId: null
                })
            get().optimisticCreateMessage(
                {
                    topicId: topicId || '',
                    title: '',
                    description: '',
                    status: 'pending',
                    role: 'assistant',
                    content
                },
                {
                    tempMessageId: temAssistantId,
                    // operationId: null
                })
            try {
                // 触发落库四连：让后端创建 Message 和 Operation (如果没有 topicId，后端会自动建 Topic)
                const res = await fetch("/api/agent/operations", {
                    method: "POST",
                    headers: { "Content-Type": "application/json" },
                    body: JSON.stringify({ content, topicId })
                });

                if (!res.ok) throw new Error(`Failed to send message: ${res.statusText}`);

                const data = await res.json();
                console.log("✅ Message operation created:", data);

                // 如果是新会话，后端返回了新生成的 topicId，此时必须把它设为激活状态
                if (!topicId && data.topicId) {
                    // 注意：你需要在 Store 里定义 activeTopicId，并在这里通过 set 把它存进去
                    // 比如: (set as any)({ activeTopicId: data.topicId });
                    console.log("🆕 New topic created! Remember to update your global activeTopicId state to:", data.topicId);
                }

                get().replaceMessages(data.messages, {
                    topicId: data.topicId
                });
                get().switchTopic(data.topicId)
                // get().replaceMessages(temAssistantId, data.assistantMessageId);

            } catch (error) {
                console.error("❌ sendMessage error:", error);
            }
        },
        reSendMessage: () => { },
        delSendMessage: () => { }
    }
}

```
