# 最小实现
执行器这边的逻辑实在过多，我们来说一下最小的实现
目标链路：

user_input → call_llm → 流式文字 → llm_result → finish

|功能|是否必须|说明|
|---|---|---|
|解析 `model` / `provider`|必须|从 payload 或 `state.modelRuntimeConfig` 取|
|创建/复用 assistant message|必须|前端要靠 `assistantMessage.id` 挂载流式内容|
|`stream_start` / `stream_end`|必须|前端知道哪条消息在生成、何时结束|
|`publishStreamChunk`（text）|必须|流式展示正文|
|调 `ModelRuntime.chat`|必须|真正请求大模型|
|基础 messages 处理|必须|至少把 `state.messages` 传给模型；可先不做复杂注入|
|`messageModel.update` 写最终 content|必须|刷新后还能看到完整回复|
|`newState.messages.push(assistant)`|必须|下一轮对话要有历史|
|返回 `nextContext: { phase: 'llm_result', ... }`|必须|Brain 才能走到 `finish`|
|错误时发 `type: 'error'` 事件|必须|前端能展示失败|
