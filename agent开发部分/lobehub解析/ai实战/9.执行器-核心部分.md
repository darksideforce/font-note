# 最小实现 🎯

执行器这边的逻辑实在太多，我们先来看一下最小实现。
目标链路：

user_input → call_llm → 流式文字 → llm_result → finish

| 功能                                             | 说明                                       | 是否必须 |
| ---------------------------------------------- | ---------------------------------------- | ---- |
| 解析 `model` / `provider`                       | 从 Payload 或 `state.modelRuntimeConfig` 中取值 | 必须   |
| 创建/复用 assistant message                        | 前端要靠 `assistantMessage.id` 挂载流式内容        | 必须   |
| `stream_start` / `stream_end`                  | 前端知道哪条消息在生成、何时结束                         | 必须   |
| `publishStreamChunk`（text）                     | 流式展示正文                                   | 必须   |
| 基础 messages 处理                                 | 最小实现直接用 `state.messages`                 | 必须   |
| 调 `ModelRuntime.chat`                          | 真正请求大模型                                  | 必须   |
| `chat` 方法传入回调                                | 直接在回调里收集内容、发送事件等                         | 必须   |
| `messageModel.update` 写最终 content              | 刷新后还能看到完整回复                              | 必须   |
| `newState.messages.push(assistant)`            | 下一轮对话要有历史                                | 必须   |
| 返回 `nextContext: { phase: 'llm_result', ... }` | Brain 才能走到 `finish`                      | 必须   |
| 错误时发 `type: 'error'` 事件                        | 前端能展示失败                                  | 必须   |

# 实现 🧩

```ts
export const createRuntimeExecutors = (ctx: RuntimeExecutorsContext) => {
    return {
        call_llm: async (instruction: AgentInstruction, state: AgentState) => {
            const { payload } = instruction;
            const { provider, model } = state
            const events: AgentType[] = []
            const { operationId, streamManager } = ctx

            // const modelRuntime = await initModelClient()
            await streamManager.publishStreamEvent(operationId, {
                data: {
                    assistantMessage: {
                        content: '',
                        id: payload.assistantMessageId,
                        role: 'assistant',
                    }
                },
                type: 'stream_start',
                stepIndex: ctx.stepIndex,
            })
            // const processMessage = payload.message
            const processMessage = [
                {
                    role: 'user', content: payload.message
                }
            ]
            const chatPayload = {
                messages: processMessage,
                model,
                provider
            }
            // 预留出解析工具
            // 预留出解析技能
            const maxAttempts = LLM_MAX_RETRIES + 1
            let bufferText = ''
            let reasonBufferText = ''
            let bufferTimer: NodeJS.Timeout | null = null
            let reasonBufferTimer: NodeJS.Timeout | null = null
            let streamError: any = null
            const flushBuffer = async () => {
                const delta = bufferText;
                bufferText = ''
                if (!!delta) {
                    events.push({
                        chunk: { text: delta, type: 'text' },
                        type: 'llm_stream',
                    });
                    await streamManager.publishStreamChunk(operationId, ctx.stepIndex, {
                        chunkType: 'text',
                        content: delta,
                    })
                }
            }
            const reasonFlushBuffer = async () => {
                const delta = reasonBufferText
                reasonBufferText = ''
                if (!!delta) {
                    events.push({
                        chunk: { text: delta, type: 'reasoning' },
                        type: 'llm_stream',
                    });
                    await streamManager.publishStreamChunk(operationId, ctx.stepIndex, {
                        chunkType: 'reasoning',
                        reasoning: delta,
                    })
                }
            }
            const modelRuntime = initModelRuntime(provider)
            for (let attempt = 1; attempt <= maxAttempts; attempt++) {
                let content = ''
                let reasonContent = ''
                streamError = null
                bufferText = ''
                reasonBufferText = ''
                const cancelBuffer = () => {
                    if (bufferTimer) {
                        clearTimeout(bufferTimer)
                        bufferTimer = null
                    }
                    if (reasonBufferTimer) {
                        clearTimeout(reasonBufferTimer)
                        reasonBufferTimer = null
                    }
                    bufferText = ''
                    reasonBufferText = ''

                }
                const response = await modelRuntime.chat(chatPayload, {
                    callback: {
                        onText: (text) => {
                            console.log(`执行了`)
                            content += text
                            bufferText += text
                            if (!bufferTimer) {
                                bufferTimer = setTimeout(async () => {
                                    await flushBuffer();
                                    bufferTimer = null;
                                }, 50);
                            }
                        },
                        onThinking: (text) => {
                            console.log(`执行了thinking`)
                            reasonContent += text
                            reasonBufferText += text
                            if (!reasonBufferTimer) {
                                reasonBufferTimer = setTimeout(async () => {
                                    await reasonFlushBuffer();
                                    reasonBufferTimer = null;
                                }, 50);
                            }
                        },
                        onCompletion: (data) => {
                            // TODO: 预留的 completion 处理逻辑
                            console.log(`执行了 onCompletion`, data);
                        },
                        onError: (error) => {
                            streamError = error
                        }
                    }
                })
                await consumeStreamUntilDone(response)
                if (streamError) {
                    throw streamError
                }
                await flushBuffer()
                await reasonFlushBuffer()
                cancelBuffer()
                await streamManager.publishStreamEvent(operationId, {
                    data: {
                        finalContent: content,
                        finalReason: reasonContent,
                    },
                    type: 'stream_end',
                    stepIndex: ctx.stepIndex,
                })
                await ctx.messageModel.update(payload.assistantMessageId, {
                    content,
                    reasoning: reasonContent ? { content: reasonContent } : undefined,
                })
                const newState = structuredClone(state)
                newState.messages.push({
                    content,
                    role: 'assistant',
                    type: 'text',
                    id: payload.assistantMessageId,
                })
                return {
                    events,
                    newState,
                    nextContext: {
                        phase: 'llm_result',
                        payload: {
                            hasToolsCalling: false,
                            result: { content, tool_calls: [] },
                            toolsCalling: [],
                        }
                    }
                }
            }


        }
    }
}
```
这个执行器很简单，它主要完成以下步骤：

1. 取出 Payload，以及 `model` 等 State 中存储的数据。
2. 取出 `operationId` 和 `streamManager` 等推送时需要的数据。
3. 先发送一次消息，通知外部即将开始 Stream 推流。
4. 组装即将提交给 `chat` 方法的数据。
5. 创建临时变量来存储数据。由于 Chunk 是流式的，每次只包含一个小事件，因此必须使用更高作用域的变量来保存它们。
6. 创建 Buffer 方法。Buffer 会在缓冲期间存储数据，直到缓冲结束后才触发提交行为并清空数据。
7. 创建若干 Callback，在 Callback 内收集数据，并使用 `streamManager` 通知外部。
8. 将 Callback 和 Payload 提交给 `chat` 方法，并等待 Response。
9. Response 返回成功后，创建 Reader 持续读取，直到 Stream 流结束。
10. 提交并清空最后的 Buffer。
11. 通知外部推流结束。
12. 将数据写入数据库。
13. 组装大脑下一次执行所需的数据，并将其返回。
