# 概述 💡

由于需要适配多个 Provider 以及模型，多个模型之间会有些许差异，而且有的 Provider 需要的字段不同，需要进行翻译；有的 Provider 拥有自己的 SDK。我们需要一层来拍平这些差异。

而且因为 OpenAI 的 SDK 返回的是可迭代对象，这并不是一个方便操作的对象，我们需要进行一次封装。

除此之外，我们还需要提供出一个通用的对象，暴露出类似 `chat`、`createImage` 等对象，方便进行操作。

所以我们需要至少三个函数：

- 拍平差异的 Provider 特有方法：它会针对当前使用的 Provider 来进行翻译。
- 针对 Stream 流的处理函数：将大模型的返回值打磨成规范输出。
- 通用的统一类（适配器）：暴露出通用方法，并且提供特殊的初始化方法来兼容各种处理函数。

## 包装 SSE 流 📌

适配器的话在上一章讲，目前我们来讨论如何规范化输出 SSE 流。

正常来说，OpenAI 等 SDK 返回给用户或者开发者的只是被包裹在 `response.body` 里的一串数据。它会在每次我们调用 `read` 方法时返回数据；如果当前客户端协议层中的缓存满了，则会暂停和大模型的通信，直到开发者继续调用 `read` 来对缓存池进行泄压。

可以看出来，`response` 是一种单向的、不可回溯的数据。这种数据如果需要调用肯定非常麻烦，所以这里需要我们来进行一次包装。

目标：

- 合理的 Stream 流。

1. 一个合理的 Stream 流首先需要针对的是一个迭代器，迭代器提供 `next` 方法，可以让 Stream 流进行可控的拉取。所以我们要保证进来的是一个迭代器。
2. 保证接收到的数据流是一个可迭代器以后，接下来我们就可以简单地对其进行包装，包装成一个 `ReadableStream` 对象。
3. 手动添加 `ReadableStream` 对象的 `pull`、`cancel`、`start` 方法。
4. 保证 `pull` 方法触发了迭代器的 `next` 方法。
5. 使用 `ReadableStream` 对象自带的 `controller` 来进行关闭和入列。
6. 当触发到迭代器触底时，执行 `close`。

```typescript
export const convertIterableToStream = <T>(
  stream: AsyncIterable<T>,
) => {
  const iterable = chatStreamable(stream); // 冗余代码
  const it = iterable[Symbol.asyncIterator](); // 转换为迭代器

  return new ReadableStream<T>({
    async cancel(reason) {
      await it.return?.(reason); // 关闭，reason 是消费者传入的
    },
    async pull(controller) {
      try {
        const { done, value } = await it.next(); // 取出迭代器返回的对象
        if (done) controller.close(); // 判断迭代器是否结束
        else controller.enqueue(value); // 入队，实际上消费者调用 read 只是拿到队列里最新的数据，每次 pull 都是根据队列里还有没有数据来进行触发的
      } catch (e) {
        controller.enqueue(e);
        controller.close();
      }
    },
    async start(controller) {
      try {
        const { done, value } = await it.next(); // start 其实也是做一样的事
        if (done) controller.close();
        else controller.enqueue(value);
      } catch (e) {
        controller.enqueue(e);
        controller.close();
      }
    },
  });
};
```

以上就是一个很简洁的 SSE 流包装。

`const it = iterable[Symbol.asyncIterator]();` 这一行是专门获取异步迭代器的固定写法。

## 包装成合理的 Streamable 消费对象 🧩

仅仅是让消费者可以合理地拿到数据还不够，我们还需要针对数据流进行格式化。

由于上一步我们已经成功创建了一个可以使用的 `ReadableStream` 对象，我们在这一步可以使用所有 Stream 对象都可以调用的 `pipeThrough()` 方法。

该方法可以接受一个 `TransformStream` 对象，以一种链式调用的方式对接收到的流进行转换，或者执行一对一的读写任务。（注意：如果流被转换，则会导致锁定，其他对象将无法读取这个流。）

`pipeThrough(transformStream)`

`pipeThrough` 很简单，只是一个入口。接下来我们需要说一下 `TransformStream` 是如何运作的。

什么是 `TransformStream` 对象？

这是一个浏览器或者 Node.js 内置的对象，通常通过 `new TransformStream` 来进行创建。它一般只用于服务 `pipeThrough` 方法，提供对管道流的操作，可以针对流进行转换或者格式化。它暴露出以下属性进行调用：

- `start(controller)`：顾名思义，就是在流开始前进行触发。
- `transform(chunk, controller)`：对每一帧流进行转换操作，暴露出的 `chunk` 就是帧对象。
- `flush(controller)`：流结束时进行触发。

简单的实例代码：

```typescript
const textEncoderStream = new TransformStream({
  transform(chunk, controller) {
    console.log('[transform]', chunk);
    controller.enqueue(new TextEncoder().encode(chunk));
  },
  flush(controller) {
    console.log('[flush]');
    controller.terminate();
  },
});
```

接下来我们就需要使用这个管道 `transformer` 方法来针对刚才封装好的流进行转换。现在大模型返回的还是 `chunk` 对象，而不是 SSE 协议可以消费的纯文本（SSE 只能接受纯文本传输）。所以我们需要将传输进来的对象转换成纯文本：

```typescript
export const createSSEProtocolTransformer = (
  transformer: (chunk: any, stack: StreamContext) => StreamProtocolChunk | StreamProtocolChunk[],
  streamStack?: StreamContext,
  options?: { requireTerminalEvent?: boolean },
) => {
  let hasTerminalEvent = false;
  const requireTerminalEvent = Boolean(options?.requireTerminalEvent);

  return new TransformStream({
    flush(controller) { // 在这里发送终结事件
      if (requireTerminalEvent && !hasTerminalEvent) {
        const id = streamStack?.id || 'stream_end';
        const data = {
          body: { name: 'Stream parsing error', reason: 'unexpected_end' },
          message: 'Stream ended unexpectedly',
          name: 'Stream parsing error',
          type: 'StreamChunkError',
        };
        controller.enqueue(`id: ${id}\n`);
        controller.enqueue(`event: error\n`);
        controller.enqueue(`data: ${JSON.stringify(data)}\n\n`);
      }
    },
    transform: (chunk, controller) => { // 在这里发送拍平了的文本 chunk 流
      const result = transformer(chunk, streamStack || { id: '' }); // 可以当作原样通过
      const buffers = Array.isArray(result) ? result : [result];
      buffers.forEach(({ type, id, data }) => {
        controller.enqueue(`id: ${id}\n`);
        controller.enqueue(`event: ${type}\n`);
        controller.enqueue(`data: ${JSON.stringify(data)}\n\n`);

        // mark terminal when receiving any of these events
        if (type === 'stop' || type === 'usage' || type === 'error') hasTerminalEvent = true;
      });
    },
  });
};
```

我们在 `transform` 方法里进行拍平。`transform` 前面讲过，是 Stream 流特有的方法，可以由管道方法注入，管道的每一帧都会被执行我们定义好的 `transform` 方法。

仔细看一下我们做了什么：实际上就是将传入的值拍成三帧塞入队列。

不仅可以简单地拍平对象做到纯文本，我们还可以让方法执行我们想要的回调，这样可以在监听到需要的事件时执行对应的回调。

```typescript
export function createCallbacksTransformer(cb: ChatStreamCallbacks | undefined) {
  const textEncoder = new TextEncoder();
  let aggregatedText = '';
  let aggregatedThinking: string | undefined = undefined;
  let usage: ModelUsage | undefined;
  let speed: ModelPerformance | undefined;
  let grounding: any;
  let toolsCalling: any;
  let streamError: any;
  let finishReason: string | undefined;
  const base64Images: Array<{ data: string; id: string }> = [];

  let currentType = '' as unknown as StreamProtocolChunk['type'];
  const callbacks = cb || {};

  return new TransformStream<string, Uint8Array>({ // 我们先只需要关注一下 transform 方法即可
    async flush(): Promise<void> {
      const data = {
        error: streamError,
        finishReason,
        grounding,
        speed,
        text: aggregatedText,
        thinking: aggregatedThinking,
        toolsCalling,
        usage,
      };

      if (callbacks.onCompletion) {
        await callbacks.onCompletion(data);
      }

      if (callbacks.onFinal) {
        await callbacks.onFinal(data);
      }
    },

    async start(): Promise<void> {
      if (callbacks.onStart) await callbacks.onStart();
    },

    async transform(chunk: string, controller): Promise<void> {
      controller.enqueue(textEncoder.encode(chunk));
      if (chunk.startsWith('event:')) {
        currentType = chunk.split('event:')[1].trim() as unknown as StreamProtocolChunk['type'];
      }
      // if the message is a data chunk, handle the callback
      else if (chunk.startsWith('data:')) {
        const content = chunk.split('data:')[1].trim();

        const data = safeParseJSON(content) as any;

        if (!data) return;

        switch (currentType) {
          case 'text': {
            aggregatedText += data;
            await callbacks.onText?.(data);
            break;
          }

          case 'reasoning': {
            if (!aggregatedThinking) {
              aggregatedThinking = '';
            }

            aggregatedThinking += data;
            await callbacks.onThinking?.(data);
            break;
          }

          case 'base64_image': {
            // data format: { image: { id, data }, images: [...] }
            const imageData = data as { image: { data: string; id: string }; images: any[] };
            base64Images.push(imageData.image);
            await callbacks.onBase64Image?.({
              image: imageData.image,
              images: base64Images,
            });
            break;
          }

          case 'content_part': {
            // data format: StreamPartChunkData
            const partData = data as StreamPartChunkData;
            await callbacks.onContentPart?.({
              content: partData.content,
              mimeType: partData.mimeType,
              partType: partData.partType,
              thoughtSignature: partData.thoughtSignature,
            });
            break;
          }

          case 'reasoning_part': {
            // data format: StreamPartChunkData
            const partData = data as StreamPartChunkData;
            await callbacks.onReasoningPart?.({
              content: partData.content,
              mimeType: partData.mimeType,
              partType: partData.partType,
              thoughtSignature: partData.thoughtSignature,
            });
            break;
          }

          case 'usage': {
            usage = data;
            await callbacks.onUsage?.(data);
            break;
          }

          case 'speed': {
            speed = data;
            break;
          }

          case 'grounding': {
            grounding = data;
            await callbacks.onGrounding?.(data);
            break;
          }

          case 'tool_calls': {
            if (!toolsCalling) toolsCalling = [];
            toolsCalling = parseToolCalls(toolsCalling, data);

            await callbacks.onToolsCalling?.({ chunk: data, toolsCalling });
            break;
          }

          case 'stop': {
            if (typeof data === 'string' && data && !finishReason) {
              finishReason = data;
            }
            break;
          }
          case 'error': {
            streamError = data;
            await callbacks.onError?.(data);
            break;
          }
        }
      }
    },
  });
}
```

需要注意的是，这段代码由于是跟在刚才那段“将对象拍成文本”的转换器之后，所以我们无法通过对象的属性来判断当前的 `chunk` 是什么类型。

所以需要创建一个跨 `chunk` 的变量，利用闭包将它保存起来。这样它可以跨 `chunk` 读取；而且由于 SSE 文本格式的特性，一定会先出现 `event` 这个文本。
